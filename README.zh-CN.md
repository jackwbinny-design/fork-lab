# Fork Lab

[English README](./README.md)

Fork Lab 是一个面向 GitHub fork 收藏的学习决策看板。

它解决的不是“怎么发现更多项目”，而是另一个更常见、也更麻烦的问题：

很多人会在 GitHub 上 fork 大量感兴趣的仓库，想着以后再研究。但时间一长，这些 fork 会慢慢变成一堆没有处理过的收藏：

- 项目很多
- 当时为什么 fork 已经忘了
- 哪些还值得看不清楚
- 每周到底该先研究哪几个也没有优先级

Fork Lab 就是针对这个问题做的。

它想把“被动收藏”变成“主动决策”：每周打开一次，看清哪些项目值得继续投入，哪些项目其实属于同一条学习路线，哪些应该留下做参考，哪些应该干脆丢掉。

## 为什么会做这个项目

这个项目最初来自一个很具体的使用场景：

一个 GitHub 账号会不断 fork 各种感兴趣的开源项目，尤其是 AI agent、设计工具、自动化流程、浏览器增强、本地部署、知识管理这几类。fork 的时候往往是因为“先留着，之后看”，但几周或几个月之后，账号里已经堆了几百个 fork，真正被认真处理的却很少。

这时候，问题已经不是“发现项目”了，而是“如何判断和取舍”：

- 我当时为什么 fork 这个？
- 这个项目现在还活着吗？
- 它到底适不适合我继续投入？
- 哪些仓库其实是同一条能力路线，应该一起看？
- 这周最值得推进的 3 组是什么？

Fork Lab 想做的，就是给这些问题一个更清晰、更可执行的界面。

## Fork Lab 是什么

Fork Lab 是一个用 React + Vite 做的前端 Dashboard，它会把 GitHub fork 列表整理成一层“决策界面”。

它整合的不是单一数据，而是几层信息：

- 仓库基础信息
- 上游项目线索
- README 摘要与重点
- 人工整理的项目关系簇
- 本地保存的每周判断与行动记录

所以它不是一个普通的仓库列表页，而更像一个轻量的个人研究工作台。

## 核心思路

Fork Lab 把 fork 收藏看成一个三段式流程：

1. 先把感兴趣的项目 fork 下来，避免链接丢失。
2. 再补充足够的上下文，帮助自己以后更快判断。
3. 最后按周复盘，让这些收藏真正变成行动，而不是继续堆积。

因此它的界面重点不是“展示全部仓库”，而是围绕这几个动作来组织：

- 精选候选
- 本周行动组
- 相关项目簇
- 留存 / 深挖 / 丢弃 决策

## 它是怎么工作的

### 1. 先把 fork 变成结构化库存

Dashboard 会从一个 GitHub 账号的 fork 仓库出发，把它们整理成可筛选、可排序的数据集。

每条记录可以包含：

- 仓库名和上游来源
- 分类和评分
- 项目描述与语言
- README 重点
- 起步动作建议
- 面向学习和判断的中文说明

### 2. 再按“项目簇”来理解，而不是单看单个仓库

很多仓库不适合孤立评估。

一个仓库可能是主项目，另外几个则可能是：

- 配套工具
- 更轻量的替代品
- 实现参考
- 数据集或提示词资料

Fork Lab 会把这些项目整理成 cluster，让你按主题复盘，而不是被随机的单个仓库打散注意力。

### 3. 最后收束到每周行动

它不会要求你把所有 fork 一次看完。

它真正想推动的是：

- 先挑少数几个强候选
- 跑一次真实测试
- 记下学到了什么
- 然后决定是继续、保留、转成产品方向，还是直接丢弃

## 适合谁

Fork Lab 比较适合这类人：

- 经常 fork 项目做研究或收集灵感的人
- 在 AI、设计、自动化、工具、副业探索这些方向来回切换的人
- 觉得浏览器书签、GitHub stars、原始 fork 列表都不够用的人
- 想把开源发现流程做成“每周可执行复盘”的人

它不是一个通用型 GitHub 数据分析平台。

更准确地说，它是一个个人学习与项目判断看板。

## 当前功能

- React + Vite 单页 Dashboard
- 仓库总览与精选候选
- 本周行动组
- 相关项目 cluster 视图
- 本地决策状态与每周记录持久化
- 上游详情富化脚本
- README 线索提取脚本
- GitHub Pages 自动部署工作流

## 本地运行

```bash
npm install
npm run dev
```

默认开发地址：

```text
http://127.0.0.1:5173
```

## 构建

```bash
npm run build
```

## 数据刷新

```bash
npm run update:data
npm run enrich:details
npm run enrich:readmes
npm run refine:categories
```

一键刷新并重建：

```bash
npm run update:dashboard
```

## 部署到 GitHub Pages

仓库内已包含 GitHub Actions 工作流：

- 推送到 `main` 后自动构建
- 自动发布 `dist/` 到 GitHub Pages

首次启用时，请在 GitHub 仓库设置里确认：

1. `Settings -> Pages`
2. `Build and deployment` 选择 `GitHub Actions`

## 技术栈

- React 18
- Vite 8
- Lucide React
- Node.js 脚本做 GitHub 数据整理

## 项目状态

这是 Fork Lab 的第一版公开 demo。

它当前只专注解决一个核心问题：把大量 GitHub fork 收藏，从“看过但没处理”变成“能复盘、能理解、能行动”的项目池。

> 如果这个项目作为公开演示仓库发布，可能会使用一份精简过的示例数据集来降低仓库体积、提升部署稳定性；本地工作区仍然可以继续使用完整数据源和更新脚本。
